프로그래머스 1단계 - 하샤드 수

🍞 문제 설명

양의 정수 x가 하샤드 수이려면 x의 자릿수의 합으로 x가 나누어져야 합니다.
예를 들어 18의 자릿수 합은 1+8=9이고, 18은 9로 나누어 떨어지므로 18은 하샤드 수입니다.
자연수 x를 입력받아 x가 하샤드 수인지 아닌지 검사하는 함수, solution을 완성해주세요.

🍞 제한 조건

x는 1 이상, 10000 이하인 정수입니다.

🍞 입출력 예

arr return
10 true
12 true
11 false
13 false
입출력 예 #1
10의 모든 자릿수의 합은 1입니다. 10은 1로 나누어 떨어지므로 10은 하샤드 수입니다.

입출력 예 #2
12의 모든 자릿수의 합은 3입니다. 12는 3으로 나누어 떨어지므로 12는 하샤드 수입니다.

입출력 예 #3
11의 모든 자릿수의 합은 2입니다. 11은 2로 나누어 떨어지지 않으므로 11는 하샤드 수가 아닙니다.

입출력 예 #4
13의 모든 자릿수의 합은 4입니다. 13은 4로 나누어 떨어지지 않으므로 13은 하샤드 수가 아닙니다.

🍞 문제 풀이

includes() 메서드로 문자의 포함 여부를 확인하고 match() 메서드와 length로 문자의 갯수를 파악하였습니다. 쉽게 하려다가 더 어렵게 해버린 기분이 드네요… 런타임 오류는 p 또는 y가 없거나 둘 다 없는 경우라네요. 주의… 

function solution(s){
    let answer = false;
    
    if(s.toLowerCase().includes("p") && s.toLowerCase().includes("y")){
        (s.match(/p/ig).length != s.match(/y/ig).length) ? answer : answer = true;    
    } else if(!s.toLowerCase().includes("p") && !s.toLowerCase().includes("y")){
        answer = true;
    }

    return answer;
}

🍞 다른 이의 풀이

펼침연산자와 reduce() 메서드, 삼항연산자를 사용하면 이런 방법도 가능하네요. 보기엔 복잡하지만 다양하게 활용할 수 있는 방법인 것 같습니다. 

function solution(s){
    return [...s.toLowerCase()].reduce((acc, cur) => {
        if(cur ==='p') return acc + 1;
        else if(cur ==='y') return acc - 1;
        return acc;
    }, 0) ? false : true;
}
